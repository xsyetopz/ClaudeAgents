Product direction:
Olympus is PiCodingAgent-first. Track A, Track B, and Track C must integrate as Olympus-owned Pi-native features.

Track A:
Safety + runtime policy.

Track B:
Authoring + workflow UX.

Track C:
Reporting + RTK-backed efficiency.

Critical RTK policy:
RTK is a first-class integration target. If RTK is available, Olympus must detect and prefer RTK-backed compaction paths for supported output-heavy workflows. Fallback compactors are allowed only when RTK is missing, unsupported, or insufficient, and fallback/degraded reason must be recorded.
