Product direction:
Olympus is PiCodingAgent-first. RTK is a first-class integration target. Use RTK when available. Do not silently avoid RTK. In-house compactors are fallback only when RTK is missing, unsupported, or insufficient.

Scope:
Implement reporting, handoff, compaction, RTK detection/status, quota profile/status, and acceptance-report aggregation inside packages/olympus.

Implement modules under packages/olympus/src as needed:
- reporting/
- compaction/
- quota/
- handoff/
- acceptance/
- status/report extensions if already appropriate
or equivalent names consistent with existing package layout.

Required features:

1. Report schema hardening
- deterministic report schemas
- package risk report
- status report
- acceptance report
- compaction report
- handoff report
- quota report
- no secrets in reports

2. Deterministic status/handoff reports
- compact handoff current state
- project-local status summary
- drift summary where available
- no stale OAL active-product claims

3. RTK detection and RTK-backed command policy
- detect RTK on PATH
- report RTK status in Olympus reports
- define RTK-backed command recommendations for output-heavy workflows
- supported command categories:
  - shell output
  - read
  - grep/find/rg
  - git diff/status/log
  - test output
  - package-manager logs
- if RTK is available, report RTK-backed path as preferred
- if RTK is unavailable, record degraded/fallback reason

4. Compaction core
- implement interfaces for compact/verbose/raw modes
- preserve decision-critical context:
  - exit status
  - failing test names
  - error messages
  - changed files
  - deleted files
  - redaction notices
  - blocked-policy reasons
  - policy warnings
  - approval decisions
- compaction must not hide safety-critical failures
- token compaction fails open only after recording fallback/degraded reason
- safety policy still fails closed

5. Fallback compactors
Implement in-house fallback compactors for:
- git diff/status/log shaped text
- test output shaped text
- grep/rg shaped text
- package-manager shaped text
- stack trace / error shaped text

These are fallback compactors, not a replacement for RTK.

6. Quota profile/status
- user-configured labels:
  - plus
  - pro-5x
  - pro-20x
  - unknown
- local usage estimate schema with actual https://github.com/Soju06/codex-lb implementation detail for ChatGPT usage tracking.
- provider-observed usage field if available
- uncertainty labels
- expensive workflow warning
- unknown quota must be labeled unknown, not fabricated
- do not hardcode opaque provider limits as truth
