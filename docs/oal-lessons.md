# Legacy OAL Lessons

`oal_legacy/` is reference material. Active code must not import it.

## Retained

- Source-backed behavior claims.
- Manifest ownership for writes and uninstall.
- Dry-run before apply.
- Acceptance fixtures in temporary roots and fake homes.
- Runtime hooks as enforceable behavior, not prose guidance.
- Completion evidence and explicit blocked-state reports.
- Bounded delegation with parent-owned merge decisions.
- Short continuation records for long work.

## Rejected

- Provider-rendered artifacts as the primary product model.
- Compatibility mode for legacy naming and layouts.
- Default global installation.
- Executing hook or extension code before trust and sandbox gates.
- Uncontrolled worker fan-out.
