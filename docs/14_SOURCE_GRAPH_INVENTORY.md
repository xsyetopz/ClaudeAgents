output.


## Runtime policy state

Policy records are present but disabled:

```json
{
  "render": { "enabled": false }
}
```

Disabled policies must not render provider hooks, scripts, or fake behavior.

## Validation boundaries

`scripts/validate-source.ts` validates:

- JSON Schema shape;
- local schema references;
- no invented schema identifier URLs;
- source record naming;
- route, skill, surface, model role, model alias, and command references;
- provider file-format facts;
- no whimsical/non-production terms in structural JSON.

It does not validate or snapshot prompt/skill prose.
